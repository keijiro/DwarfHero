# プロジェクト概要
- ゲームタイトル: Match-3 Combat Hybrid (マッチ3・コンバット・ハイブリッド)
- 目的: 現在ハードコードされているゲームバランス（成長曲線、敵ステータス、ウェーブ予算、ブロック消去計算式等）を `ScriptableObject` に移行し、インスペクター上でシミュレーション・調整が可能なエディターツールを構築する。

# ゲームメカニクス (調整・管理対象)
- **ブロック消去計算式**: マッチ数とSkaブロック（お邪魔ブロック）の寄与率を決定する係数。
- **成長曲線**: プレイヤーのLvごとのHP/ATK、次レベルへの必要経験値。
- **アクション効率**: ブロック1個あたりの回復量、シールド獲得量（最大HPへの到達速度）。
- **敵ステータス**: 各種モンスターのHP、攻撃力、攻撃間隔、コスト。
- **難易度推移**: ウェーブごとの予算、および隊列による減衰率。

# UI (エディターツール)
- **カスタムインスペクター**: `ScriptableObject` を選択した際、以下の情報を表示する。
    - **消去計算シミュレーター**: 「3マッチ + Ska 3個で有効カウントがいくつになるか」のプレビュー。
    - **Lv別ステータス表**: プレイヤーの成長を一覧表示。
    - **戦闘バランス分析**: 各モンスターに対し「何回殴れば倒せるか」「何回攻撃されたら死ぬか」の分析表示。
    - **ウェーブプレビュー**: 指定したウェーブにおける予算と、出現モンスターのパターンのシミュレーション。

# 主要アセットとコンテキスト
- `GameBalanceData.cs`: パラメータを保持する `ScriptableObject`。
- `GameBalanceDataEditor.cs`: インスペクターの表示をカスタマイズするエディタークラス。
- `CombatManager.cs`: `GameBalanceData` を参照するように変更。
- `EnemyUnit.cs`: `CombatManager` を介してバランスデータを利用。

# 実装ステップ

## 1. `ScriptableObject` の定義 (`GameBalanceData.cs`)
- 以下のカテゴリに分けたパラメータを定義する。
    - **MatchingRules**: Skaブロックの寄与係数（例：`skaDivisor = 3`）。
    - **PlayerStats**: 初期値、Lvごとの増加量（HP, ATK, XP）。
    - **Actions**: 回復倍率、シールド分母。
    - **Enemies**: `EnemyDefinition` のリスト。
    - **Wave**: 初期予算、増加率、隊列減衰率。

## 2. カスタムインスペクターの実装 (`Editor/GameBalanceDataEditor.cs`)
- `OnInspectorGUI` をオーバーライドし、以下のシミュレーション機能を追加する。
    - **計算式のプレビュー**: 設定した係数に基づき、実際のアクション値がどう計算されるかを表示。
    - **分析テキスト**: 「スライムを倒すのに必要な攻撃回数」などを表示。
    - **レベル推移表**: 1〜20レベルまでの主要ステータスを一覧表示。

## 3. `CombatManager` の統合
- `AddPlayerAction` メソッド内の `effectiveCount` 計算式を `balanceData` 参照に修正。
- その他、HP/ATK/EXP/Action値の計算を `balanceData` 参照に統合。

## 4. `EnemyUnit` の調整
- 隊列減衰率を `balanceData` から取得するように修正。

## 5. データの移行と検証
- 現在のバランス値を `ScriptableObject` に反映し、動作を確認。

# 検証とテスト項目
- **計算式反映の確認**: `skaDivisor` を変更した際、Skaブロックを消した際のアクション増加量が正しく変化するか確認。
- **データ反映確認**: `ScriptableObject` の値を書き換えた際、即座にゲーム内のダメージ量やレベルアップ必要数が変化することを確認。
