# プロジェクト概要
- ゲームタイトル: Match-3 RPG Prototype
- コンセプト: 7x7のグリッドでブロックを消して戦闘アクションをトリガーする2Dマッチ3 RPG。
- プレイヤー: シングルプレイヤー
- レンダリングパイプライン: URP 2D
- 画面の向き: 横向き (Landscape)

# ゲームメカニクス
## コアゲームプレイ
プレイヤーはグリッド上のブロックをマッチさせます。マッチにより攻撃、回復、シールドなどのアクションが実行されます。敵はタイマーに基づいて攻撃してきます。敵のウェーブを突破していくことでゲームが進行します。

## 進行の明確化（ファンタジー演出の強化）
以下の視覚的演出とウェーブ間の区切りを追加し、ゲームの進行を分かりやすくします。
1. **開始時の待機**: ゲーム開始時に1秒間の間を置く。
2. **登場メッセージ**: 敵が登場する前に、ファンタジーRPG風のメッセージを表示する（例: "MONSTERS APPROACH!"）。
3. **順次登場**: 敵を一体ずつ順番に表示し、その際に白くフラッシュさせる。
4. **全滅メッセージ**: 敵パーティー全滅時に勝利メッセージを表示する（例: "VICTORY!"）。
5. **ウェーブ間待機**: 次のウェーブが始まるまでに1秒間の間を置く。
6. **ゲームオーバー演出**: HPが0になった際、パーティ全員を赤フラッシュさせ、その後キャラクターを黒塗り（シルエット）にし、1秒待機してからゲームオーバー画面へ遷移する。

# UI
## 中央メッセージ
`Main.uss` に、画面中央に大きく表示されるラベル用のスタイルを追加します。
- `.center-message`: 大きなフォント（Pirata Oneを使用）、中央配置、ファンタジー風の装飾。フェードイン/アウトのアニメーション。

# 主要アセットとコンテキスト
- `CombatManager.cs`: ゲームフローと戦闘アクションを管理するメインコントローラー。
- `CharacterVisuals.cs`: スプライトのフラッシュやエフェクトを制御するコンポーネント。
- `Main.uss`: UI Toolkit用のスタイルシート。

# 実装手順

## 1. UIスタイルの追加
### `Assets/UI/Main.uss` の修正
- 中央メッセージ用の `.center-message` クラスを追加。
- フォントサイズを大きくし、ファンタジー感を出すために縁取りやシャドウを調整。
- 出現時と消失時のアニメーションを定義。

## 2. キャラクター演出機能の拡張
### `Assets/Scripts/CharacterVisuals.cs` の修正
- `SetColor(Color color)` メソッドの追加: 指定した色でスプライトを上塗りし、リセットされるまで保持する機能（死亡時の黒塗りに使用）。
- 継続的な色が設定された際、実行中のフラッシュ用コルーチンを停止するように修正。

## 3. コンバットマネージャーの制御
### `Assets/Scripts/CombatManager.cs` の修正
- **中央メッセージ表示**: UITKを使用してメッセージを表示する `ShowCenterMessage(string text, Color color)` を実装。
- **開始シーケンス**:
    - `Start()` をコルーチンに変更。
    - 1秒の待機と "MONSTERS APPROACH!" の表示ロジックを実装。
- **順次スポーン**:
    - 敵を一体ずつ生成する `SpawnWaveSequentially()` を実装。
    - 各敵の生成時に `CharacterVisuals` を通じて白いフラッシュをトリガー。
- **ウェーブ遷移**:
    - `CleanupEnemies()` と `SpawnWaveWithDelay()` を更新し、"VICTORY!" の表示と1秒の待機を組み込む。
- **ゲームオーバーシーケンス**:
    - `HandleEnemyAttack()` 内の死亡判定処理を更新。
    - 赤フラッシュ -> キャラクター黒塗り -> 1秒待機 -> シーン遷移の順で実行するように修正。

# 検証とテスト
## 手動テスト
1. **ゲーム開始**: 開始時に1秒の待ちがあり、"MONSTERS APPROACH!" が表示されてから敵が登場することを確認。
2. **敵の登場**: 敵が一体ずつ順番に白フラッシュを伴って現れることを確認。
3. **ウェーブクリア**: 敵を全滅させた際、"VICTORY!" が表示され、1秒後に次のウェーブが始まることを確認。
4. **ゲームオーバー**: HPが0になった際、赤フラッシュの後にメンバーが真っ黒になり、1秒後に遷移することを確認。
