# プロジェクト概要
- ゲームタイトル: Match-3 Combat Hybrid (マッチ3・コンバット・ハイブリッド)
- コンセプト: プレイヤーがグリッドの最下段を操作して重力による連鎖を引き起こし、それによって発生するアクションでモンスターと戦うリアルタイム・マッチ3パズル。
- プレイヤー: シングルプレイヤー
- レンダーパイプライン: URP
- ターゲットプラットフォーム: PC / スタンドアロン

# ゲームメカニクス
## コアゲームプレイループ
プレイヤーは7x7グリッドの最下段をクリックしてブロックを消します。ブロックが消えると上が落ちてきて、連鎖が発生します。マッチしたブロックの種類に応じて、攻撃、魔法、回復、シールド、経験値などのアクションが発生します。戦闘はリアルタイムで進行し、敵は独自のタイマーで攻撃してきます。

## RPG成長要素
- **敵のスケーリング**: 敵パーティーは「合計レベル予算」に基づいて編成されます。この予算は敵を倒すごとに（ウェーブごとに）線形に増加します。
- **プレイヤーのステータス**: レベルアップにより、最大HP、物理攻撃力、魔法攻撃力が上昇します。
- **シールドの成長**: ブロックを10個消すと現在の最大HPと同量のダメージを相殺できる値を基本単位とします。
- **経験値ソース**: 敵の撃破、Gemブロックのマッチ、宝箱の開封（要・鍵）から経験値が得られます。

## 敵のAIと隊列
- **隊列による頻度変化**: 前方にいる敵ほど攻撃頻度が高く、後方にいる敵は射程や手数の制約をシミュレートして攻撃頻度が下がります。
- **魔法攻撃の例外**: 魔法攻撃を行う敵は、隊列に関わらず常に本来の頻度で攻撃を行います。

# UI
- 既存のUITK（UI Toolkit）によるHUDを使用し、更新されたHPや経験値を表示します。
- レベルアップ通知やダメージ数値などの演出は、新しいステータス値を反映するように調整されます。

# 主要アセットとコンテキスト
- `CombatManager.cs`: ステータス管理、ウェーブ生成、アクション実行の中枢。
- `EnemyUnit.cs`: 各敵個体のロジック、タイマー、ステータス保持。
- `GridManager.cs`: ブロック破壊の処理と、マッチ発生時のCombatManagerへの通知。

# 実装ステップ

## 1. プレイヤーステータスとレベルアップロジック (`CombatManager.cs`)
- `GetMaxHPForLevel(int level)`の実装: $100 + \frac{100}{9}(level - 1)$。
- `GetAttackForLevel(int level)`の実装: $10 + 2(level - 1)$。
- `GetMagicAttackForLevel(int level)`の実装: $Attack / 3$。
- `GetThresholdForLevel(int level)`の実装: 初期値40、レベルごとに約13ずつ増加する線形成長 ($Req(L) = 40 + 13(L-1)$)。
- `AddPlayerAction`の更新: レベルに応じたアクション値を計算。
    - **シールド**: ブロック1個あたり `MaxHP / 10` （整数化、最小値1）。
    - **回復**: 攻撃力 $\times 0.6$ （整数化、最小値1）。
- `AddExperience`の更新: Gemと宝箱の経験値を動的に計算。
    - `GemEXP = Max(1, Req(Level)/10)`
    - `ChestEXP = Req(Level)/4`

## 2. 敵の定義と予算システム (`CombatManager.cs`)
- `EnemyData`構造体の定義: HP、攻撃力、攻撃間隔、レベル、魔法属性。
- 5種類の敵タイプを設定:
    - **Slime**: Lv 2, HP 30, ATK 4, CD 3s.
    - **Skeleton**: Lv 3, HP 40, ATK 4.5, CD 3s.
    - **ZombieMage**: Lv 6, HP 40, ATK 4, CD 3s, Magic.
    - **Orc**: Lv 7, HP 75, ATK 5.4, CD 3s.
    - **Golem**: Lv 10, HP 150, ATK 8.1, CD 3s.
- ウェーブ予算の実装: $最大合計レベル(Wave) = 6 + 1.2 \times Wave$。
- `SpawnWaveSequentially`の更新:
    - 残り予算に収まる敵をランダムに選出。
    - `ActiveEnemies`リスト内での並び順を管理（前方＝インデックス0）。

## 3. 敵のAIと攻撃頻度 (`EnemyUnit.cs`)
- `CombatManager`からステータスを受け取るように更新。
- 隊列（インデックス）に基づく攻撃頻度の計算を実装:
    - `CombatManager.Instance.ActiveEnemies`内でのインデックスを取得。
    - 物理攻撃の場合: $有効間隔 = 基本間隔 / 0.75^{index}$。
    - 魔法攻撃の場合: $有効間隔 = 基本間隔$。
- `Die()`の更新: `5 * Level` の経験値をプレイヤーに付与。

## 4. 検証とテスト
- **バランス確認**: レベル1でスライムを3回で倒せるか確認。
- **シールド確認**: シールドブロックを10個消すと最大HP分（Lv 1なら100）溜まるか確認。
- **ウェーブ確認**: ウェーブ15で予算が約24になるか確認。
- **レベルアップ確認**: スライム4体でレベル2になるか確認。

# 検証とテスト項目
- **実機プレイテスト**: ゲーム開始時、スライムのHP/攻撃力が設計通り（3回撃破/25回全滅）か確認。
- **シールド成長確認**: レベル10付近で、シールドブロック1個あたりの増加量が約20になっていることを確認。
- **隊列テスト**: 3体の敵が並んだ際、後方の敵の攻撃が物理攻撃なら遅くなり、魔法攻撃なら遅くならないことを確認。
